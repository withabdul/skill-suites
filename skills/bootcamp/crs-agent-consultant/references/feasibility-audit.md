# Feasibility Audit

Evaluate whether a proposed course idea is logically feasible — timeline vs. content vs. audience vs. resources. No enthusiasm unchecked, no scope unmeasured.

## What Success Looks Like

The owner receives a clear-eyed assessment: what's realistic, what's ambitious-but-possible, and what's likely to fail. The audit doesn't kill ideas — it sizes them accurately so the owner can make informed decisions.

## Your Approach

### Audit Dimensions

Check each proposed course against these dimensions:

| Dimension | What to Evaluate |
|-----------|-----------------|
| **Scope vs. Timeline** | Can the proposed content fit the proposed duration? How many topics per session? |
| **Audience Match** | Does the difficulty level match the target audience's assumed skill level? |
| **Resource Reality** | Does the owner have the expertise, time, and tools to deliver this? |
| **Market Differentiation** | What already exists? Is there genuine demand or is it assumed? |
| **Content Density** | Is there enough substance for a full course, or is this better as a tutorial or blog post? |

### Red Flags to Watch For

- "Everyone needs this" — usually a sign the audience is too broad
- "It'll just take a few hours" — scope underestimation
- "We'll figure out the details later" — the details ARE the course
- No clear "one thing" the student walks away knowing
- Timeline set before content is understood

### Green Flags Worth Noting

- Clear specific audience ("junior React devs transitioning to Next.js" not "developers")
- Measurable learning outcome ("can deploy a full-stack app" not "understands full-stack")
- Owner has demonstrable expertise in the domain
- Realistic scope that leaves breathing room

### Report Format

Present the audit as:
1. **Overall Verdict** — Feasible / Ambitious / Needs Rework / Not Viable
2. **Strong Points** — What's working in the proposal
3. **Concerns** — Specific issues with evidence
4. **Recommended Adjustments** — Concrete changes that address concerns
5. **Go/No-Go Condition** — What needs to be true for this to work
